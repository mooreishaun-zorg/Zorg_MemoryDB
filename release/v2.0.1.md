# Zorg MemoryDB v2.0.1

This full package release removes historical release artifacts from the active
source and archive surfaces. It keeps only current install dependencies and
canonical PostgreSQL-backed behavior.

## Cleanup

- Removes superseded release notes and tracked release archives from the active
  repository tree.
- Archives only the current release note, never the historical `release/`
  directory.
- Makes legacy `memory/**/*.md` import explicit with
  `ZORG_IMPORT_RETIRED_MEMORY=1`; clean installs do not import it.
- Keeps the required bootstrap/recovery rules, PostgreSQL schema and recall
  tooling, LAN Command Chat gauges, Memory 3D, installer, and verification
  surfaces.

## Verification

- Fresh install source inventory contains no historical release notes or
  release archives.
- Archive contains exactly one release note and no generated/private artifacts.
- LAN Command Chat build and TypeScript checks pass.
- Memory 3D syntax, package scan, shell checks, and PostgreSQL/Zorg MemoryDB
  health checks pass.
